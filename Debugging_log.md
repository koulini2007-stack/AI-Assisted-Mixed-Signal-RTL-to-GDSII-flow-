# AI-Assisted Debugging Log

## Overview

During the implementation of the AI-assisted mixed-signal RTL-to-GDSII flow, several issues were encountered while integrating the digital SPI controller, analog hard macro, and OpenLane physical design flow. AI-assisted prompts were used to analyze the errors, identify their root causes, and recommend appropriate fixes.

---

# Error 1 – Liberty File Threshold Definitions Missing

## Tool

OpenSTA (invoked by OpenLane)

## Error

During the floorplanning stage, OpenSTA generated multiple warnings indicating missing threshold definitions in the generated Liberty timing model.

Example:

```
output_threshold_pct_rise not found
input_threshold_pct_fall not found
slew_lower_threshold_pct_rise not found
Library AMUX2_3V.lib is missing one or more thresholds
```

## Root Cause

The AI-generated Liberty file contained valid pin definitions and timing arcs but lacked mandatory library threshold parameters required by OpenSTA.

## AI Prompt Used

```
Analyze the OpenSTA error indicating missing threshold values in the generated Liberty timing model. Explain the missing parameters and modify the Liberty file so that it becomes compatible with OpenSTA and OpenLane while preserving placeholder timing values.
```

## Solution

Added the following library-level parameters:

- input_threshold_pct_rise
- input_threshold_pct_fall
- output_threshold_pct_rise
- output_threshold_pct_fall
- slew_lower_threshold_pct_rise
- slew_upper_threshold_pct_rise
- slew_lower_threshold_pct_fall
- slew_upper_threshold_pct_fall

## Result

The Liberty parsing errors were resolved and OpenSTA successfully proceeded to the next stage.

# Error 2 – PDN Generation Failure

## Tool

OpenROAD (PDN Generator)

## Error

During floorplanning the flow terminated with the following message:

```
Pitch 6.4400 is too small.
Must be at least 6.6000
```

## Root Cause

The default PDN pitch configured by OpenLane was smaller than the minimum value supported for the generated floorplan.

## AI Prompt Used

```
Analyze the PDN generation failure during the OpenLane floorplanning stage. Explain why the PDN pitch is considered invalid and recommend modifications to the OpenLane configuration to satisfy the minimum pitch requirement.
```

## Solution

Modified the PDN configuration by increasing the PDN pitch values in the OpenLane configuration.

Additional adjustments included reducing the floorplan utilization to provide more routing resources.

## Result

Power Distribution Network generation completed successfully.

# Error 3 – Macro Placement Configuration Missing

## Tool

OpenLane

## Error

The flow stopped during IO placement with the following error:

```
error copying ""
MACRO_PLACEMENT_CFG
No such file or directory
```

## Root Cause

The OpenLane configuration referenced a macro placement configuration file that did not exist.

## AI Prompt Used

```
Analyze the macro placement configuration error generated by OpenLane. Explain why the MACRO_PLACEMENT_CFG file is required and generate a valid macro.cfg file for the analog hard macro.
```

## Solution

Created:

```
macro.cfg
```

Specified the analog hard macro instance name, placement coordinates, and orientation.

Updated the OpenLane configuration to reference the generated file.

## Result

Macro placement completed successfully.

- EXTRA_GDS_FILES

# Error 5 – LVS Net Mismatch

## Tool

Netgen

## Error

The final LVS report indicated:

```
Device count difference = 0

Net count difference = 2

Circuit 1 contains 33 nets

Circuit 2 contains 31 nets
```

## Root Cause

The digital implementation matched successfully, however the analog hard macro abstraction used during synthesis differed slightly from the extracted layout netlist, producing a small net mismatch.

## AI Prompt Used

```
Analyze the final LVS report showing a net count mismatch between the generated layout and schematic. Explain the possible causes of the mismatch and recommend debugging strategies for mixed-signal hard macro integration.
```

## Solution

Verified:

- Device count matched.
- Cell instances matched.
- Analog macro instantiation matched.
- LEF, Liberty and GDS references were correctly included.

The remaining mismatch was attributed to differences between the abstract analog model and the extracted physical layout.

## Result

LVS execution completed successfully with only a minor net mismatch remaining for the analog hard macro.

# Error 6 – Docker Container Not Found

## Tool

WSL2 / Docker

## Error

```
Exception:
No container engine found.

The command 'docker' could not be found in this WSL 2 distro.
```

## Root Cause

Docker Desktop integration with WSL2 was not enabled.

## AI Prompt Used

```
Analyze the OpenLane Docker startup error in WSL2. Explain why Docker cannot be detected and provide the steps required to enable Docker Desktop integration with WSL2.
```

## Solution

Enabled:

- Docker Desktop
- WSL2 Integration
- Ubuntu distribution under Docker Desktop settings

Restarted Docker and executed:

```
make mount
```

## Result

OpenLane container launched successfully.
